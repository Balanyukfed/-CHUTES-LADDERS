Поля ввода

Позволяют пользователям вводить текстовые данные в произвольной форме, как длинные, так и короткие тексты. Поля ввода нужно использовать, когда нет возможности предложить готовый список для выбора.

Тип используемого текстового поля должен отражать длину контента, который будет в него вводить пользователь. Текстовые области в основном используются в формах и диалогах. Высота текстовой области автоматически увеличивается при вводе текста.

Одиночное текстовое поле

Используется для короткого однострочного содержимого.

Текстовая область

Используется для более длинных многострочных записей.

Параметры

Размер поля может подсказать пользователю в какой форме предоставлять данные. Стандартная высота полей ввода - 48px, использутся в большинстве случаев.

Поведение

Вспомогательный текст

Пояснительный текст дает больше информации и контекста о вводимых данных. Он находится под полем и помогает пользователям правильно его заполнять. Часто в данном тексте прописывается объяснение требований к полю или пример правильного заполнения.

Проверка заполнения

В поля встроены комментарии для поддержки обратной связи при вводе данных. Состояния проверки визуально изменяет поле ввода, например, при ошибке появляется соответсвующая надпись и цвет поля. Также помимо отрицательных есть подтверждающие сообщения.

Руководство

Надписи должны кратко и ясно выражать информацию, которую нужнно ввести пользователю. Лучше избегать длинных предложений и не использовать глаголы в повелительном наклонении. Название в поле должно быть видно полностью, после него никаких знаков препинания ставить не нужно.
